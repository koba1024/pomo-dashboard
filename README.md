# pomo-dashboard

作業量の記録と可視化を PC 上で完結させるために開発した作業管理アプリです。  
単なるポモドーロタイマーではなく、作業対象を選んだうえでセッションを記録し、自分の足跡を振り返れるようにしています。

## 作成背景

これまで学習記録はスマートフォンのアプリで管理していました。  
しかし、記録のためにスマートフォンを触ると SNS など別アプリに逸れてしまったり、休憩中までスマートフォンを見てしまったりして、集中を切らしやすいことが課題でした。

そこで、**何のために作業するのか** と **どのくらい作業したのか** を PC 上で確認できるようにするため、本アプリを作成しました。  
作業時間だけでなく、まず着手した回数や継続の足跡も確認できる形を目指しています。

## 主な機能

- **作業対象を選んで始めるタイマー** — 学習カード / Todo を切り替えて作業対象を選択し、作業時間・休憩時間を設定してタイマーを開始できます
- **柔軟な時間設定** — 作業時間・休憩時間は固定ではなく追加でき、一度作成した設定は保存されるため次回以降も再利用できます
- **バックグラウンド対応** — 他のタブや別アプリを使用中でもタイマーが大きくずれないようにし、フェーズ切り替え時にはブラウザ通知を送信します
- **セッション記録** — 作業対象名・作業時間・開始時刻・終了時刻を保存し、後から振り返れるようにしています
- **ダッシュボード** — 連続日数、ポモドーロ数、直近7日間の作業時間などを確認できます
- **認証** — メールアドレス登録 / ゲストログインに対応しています

## このアプリで重視したこと

### 1. ただの時間ではなく、目的のある記録にすること

タイマー開始前に作業対象を選べるようにし、作業時間を単なる数字で終わらせず、**何に使った時間なのか** を後から追えるようにしています。  
学習カードは長期的に取り組むテーマ、Todo は単発の具体的な作業として扱っています。

### 2. 着手した事実を残すこと

長時間できた日だけを成功とすると、継続のハードルが上がると感じていました。  
そのため本アプリでは、作業時間だけでなく「ポモドーロ数」として着手回数も確認できるようにしています。短いセッションでも、まず始めたことを記録として残せる設計を意識しました。

### 3. 継続を実感しやすくすること

連続日数や直近7日間グラフを確認できるようにし、自分の足跡を見返せるようにしています。  
特に連続日数は、「ここで途切れさせたくない」と感じやすく、少しでも着手する動機につながると考えています。

## 技術的なポイント

### タイマー精度の改善

初期実装では `setInterval` による単純なカウントダウンを採用していましたが、タブ切り替え後に戻ると表示がずれる問題がありました。  
そこで、残り時間を直接減らすのではなく、開始時刻と `Date.now()` の差から経過時間を再計算する方式に変更しました。

この方式では、一時停止後の再開時に基準時刻を適切に更新しないと時間が一気に進んでしまうため、一時停止・再開時の状態管理も見直しています。

### 状態管理の整理

初期段階では `useState` を個別に増やしていたため、UI の状態とタイマー進行に関わる状態が散らばり、可読性や保守性が下がっていました。  
そこで、UI 状態（ダイアログ開閉、選択中タブ、入力値など）と、タイマー状態（残り時間、ステータス、フェーズ遷移など）を分けて整理しました。

その結果、機能追加や不具合修正の際に、どこを触ればよいかを判断しやすくなりました。

### 認証とデータ保存

Supabase を利用し、認証とデータベースをまとめて扱っています。  
学習カード、Todo、ポモドーロ時間設定、セッション記録をユーザーごとに保存し、RLS によりログインユーザー本人のデータのみ操作できるようにしています。

また、ゲストログインでも主要機能を一通り試せるようにし、ポートフォリオとして確認する側のハードルを下げています。

## 今後の改善予定

- 作業対象を未選択のまま開始できないようにし、記録に必ず目的が紐づくよう改善する
- 学習カードごとの累計作業時間を表示し、どのテーマにどれだけ取り組んだかを振り返れるようにする
- 学習カードを親、Todo を子とする構造へ見直し、長期テーマと短期タスクの関係をより自然にする
- ユニットテストを追加し、タイマー状態遷移まわりの挙動を検証しやすくする
